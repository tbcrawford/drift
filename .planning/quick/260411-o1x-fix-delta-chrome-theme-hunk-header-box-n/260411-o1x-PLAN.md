<objective>
Fix the DeltaTheme hunk header box rendering in non-pager modes (single-file git diff vs HEAD).
Currently `DeltaTheme.RenderHunkHeader` returns `""` (falling back to `@@ -N,n +N,n @@` format)
because `h.CodeFragment` is always empty outside pager mode. The fix: run `git diff HEAD -- <path>`
via exec subprocess, parse the `@@ ... @@ fragment` headers using the existing `parseHunkHeader`
function, and backfill `result.Hunks[i].CodeFragment` by `NewStart` match before rendering.

Purpose: Users running `drift --chrome delta myfile.go` see the Unicode box (`┐`/`•`/`┘`) instead
of the plain `@@` fallback, matching the pager-mode behavior.

Output:
- `cmd/drift/gitworktree.go` — new `gitHunkFragmentsForFile` helper
- `cmd/drift/flags.go` — add `contextLines int` field to `rootOptions`
- `cmd/drift/main.go` — backfill CodeFragment in the single-file non-pager path
- `cmd/drift/main_test.go` — new test `TestRunCLI_pagerMode_chromeDelta_codeFragment`
</objective>

Add the following function at the bottom of `cmd/drift/gitworktree.go`, after the `gitFilePair` type:

```go
// gitHunkFragmentsForFile runs "git diff --unified=N HEAD -- <path>" and returns
// a map of newStart → codeFragment parsed from the @@ hunk headers.
// This is best-effort: returns nil silently on any error (git not found, not a
// git repo, no diff, file not tracked, etc.) — never blocks rendering.
func gitHunkFragmentsForFile(path string, contextLines int) map[int]string {
    cmd := exec.Command("git", "diff",
        fmt.Sprintf("--unified=%d", contextLines),
        "HEAD", "--", path)
    out, err := cmd.Output()
    if err != nil || len(out) == 0 {
        return nil
    }
    frags := make(map[int]string)
    for _, line := range strings.Split(string(out), "\n") {
        if strings.HasPrefix(line, "@@ ") {
            _, newS, cf, ok := parseHunkHeader(line)
            if ok && cf != "" {
                frags[newS] = cf
            }
        }
    }
    if len(frags) == 0 {
        return nil
    }
    return frags
}
```

Add `"os/exec"` to the import block (it currently imports `"errors"`, `"fmt"`, `"os"`, `"path/filepath"`, `"slices"`, `"strings"` — append `"os/exec"` to this group). The function uses `parseHunkHeader` which is in the same `package main`.

**Why exec instead of go-git**: go-git does not expose function-context lines from unified diff headers — those are computed by git's diff driver using `xdiff`. We cannot replicate this without significant algorithmic work. `exec.Command("git", ...)` is best-effort and fails gracefully (nil return), matching the pattern used elsewhere in the codebase (Phase 27 decision: standalone `drift.Diff()` path unchanged; CodeFragment always '' for non-pager; pager mode uses git stdout — this extends that to single-file mode with the same best-effort approach).

// Backfill CodeFragment from git diff header lines when the input is a real
// git file (not --from/--to or stdin). Best-effort: gitHunkFragmentsForFile
// returns nil when git is unavailable or the file has no fragment context.
headerName := fileHeaderName(opts.args)
if headerName != "" && len(opts.args) == 1 {
    frags := gitHunkFragmentsForFile(opts.args[0], opts.contextLines)
    for i := range result.Hunks {
        if cf, ok := frags[result.Hunks[i].NewStart]; ok {
            result.Hunks[i].CodeFragment = cf
        }
    }
}

var buf bytes.Buffer
if headerName != "" {
    buf.WriteString(opts.chromeTheme.RenderFileHeader(headerName, opts.noColor, opts.termWidth))
}
if err := drift.RenderWithNames(result, &buf, oldName, newName, opts.driftOpts...); err != nil {
    return newExitCode(2, err.Error())
}

return writeThroughPager(&buf, opts)
```

**Key design decisions:**
- Match by `result.Hunks[i].NewStart` (not positional index) — `NewStart` is the source-of-truth line number from git's unified diff and from drift's independent hunk computation. Positional match is fragile when hunk counts differ.
- Only backfill when `len(opts.args) == 1` (single git file path). The two-file diff path (`len(opts.args) == 2`) has no git context to query — the files may not be in a git repo.
- `gitHunkFragmentsForFile` is nil-safe: if git is not available, no change is made, rendering falls through to `@@ @@` format gracefully.

// TestRunCLI_pagerMode_chromeDelta_codeFragment verifies that --chrome delta
// in pager mode with a @@ code_fragment renders the DeltaTheme box (┐, •, ┘)
// instead of the plain "@@ ... @@" fallback.
// "parseXYZ" appears only in the @@ header (not in content lines), so it can
// only reach the output via CodeFragment backfill → DeltaTheme box rendering.
func TestRunCLI_pagerMode_chromeDelta_codeFragment(t *testing.T) {
    var out, errOut bytes.Buffer
    streams := IOStreams{
        In:  strings.NewReader(gitDiffWithCodeFragment),
        Out: &out,
        Err: &errOut,
    }
    code := runCLI(streams, []string{"--no-color", "--chrome", "delta"})
    if code != 1 {
        t.Fatalf("expected exit 1 for differing input, got %d; stderr=%q stdout=%q",
            code, errOut.String(), out.String())
    }
    stdout := out.String()
    // DeltaTheme box: top-right corner character appears when codeFragment != "".
    if !strings.Contains(stdout, "┐") && !strings.Contains(stdout, "• ") {
        t.Errorf("expected DeltaTheme box characters (┐ or '• ') in rendered output, got:\n%s", stdout)
    }
    // Must NOT fall through to plain @@ format.
    if strings.Contains(stdout, "@@ -") {
        t.Errorf("rendered output should not contain @@ fallback when DeltaTheme box is active, got:\n%s", stdout)
    }
    // Code fragment must appear in the box content line.
    if !strings.Contains(stdout, "parseXYZ") {
        t.Errorf("expected 'parseXYZ' code_fragment in DeltaTheme box, got:\n%s", stdout)
    }
}
```
