# Quick Task 260401-k11: switch to single module build - Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Task Boundary

Collapse the two-module layout (root library module + cmd/drift sub-module linked by go.work) into a single go.mod at the repo root. The CLI entry point stays at cmd/drift/ but is no longer a separate module.

</domain>

<decisions>
## Implementation Decisions

### CLI location
- cmd/drift/main.go — standard Go convention, go install github.com/tbcrawford/drift/cmd/drift@latest remains valid

### Workspace files
- Delete go.work and go.work.sum entirely — they only existed to link two modules
- Delete cmd/drift/go.mod and cmd/drift/go.sum — no longer a separate module

### CLI import path for library
- CLI imports library as github.com/tbcrawford/drift — direct, no replace directive needed since it's all one module

### the agent's Discretion
- Merge dependencies from cmd/drift/go.mod into root go.mod (de-duplicate; keep highest version of any shared dep)
- Update all CI workflows that reference module boundaries (go.work, separate module test steps)
- Update security.yml — go-package can revert to ./... once the replace directive is gone

</decisions>

<specifics>
## Specific Ideas

- go install github.com/tbcrawford/drift/cmd/drift@latest must continue to work after the merge
- The replace directive in cmd/drift/go.mod (replace github.com/tbcrawford/drift => ../..) disappears — no longer needed
- CI: the "Test (CLI sub-module explicit)" step that cd's into cmd/drift can be removed; a single go test -race ./... from root covers everything

</specifics>
