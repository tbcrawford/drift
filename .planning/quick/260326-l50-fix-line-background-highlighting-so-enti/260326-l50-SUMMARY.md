# Quick Task 260326-l50: Fix Full-Line Background Highlighting

**Date:** 2026-03-26
**Status:** Complete

## What Was Done

Fixed GitHub PR-style full-line background highlighting so that entire delete (red) and insert (green) lines are highlighted, matching terrasort's approach.

## Root Cause

Two separate gaps caused incomplete line backgrounds:

1. **Prefix character unstyled** — the `+`/`-` prefix was emitted as a plain character with no background color, leaving the first column uncolored.

2. **Trailing whitespace unstyled** — chroma emits `\033[0m` (reset) at the end of each token, which clears the background before the next character. Using `lipgloss.NewStyle().Width(N)` for padding after highlighted tokens produced trailing spaces with *no* background (the reset killed it).

## Fix

Followed terrasort's `renderContent` approach:

### New helpers in `internal/render/wordline.go`
- **`styledDiffPrefix(op, bg)`** — returns the `+`/`-`/` ` prefix with the line background applied when `bg` is set.
- **`padLineBackground(content, width, bg)`** — measures `lipgloss.Width(content)`, then appends explicitly background-colored spaces to fill the remaining `width`. This is the key fix for trailing whitespace.
- **`renderPanelContent(content, panelWidth, bg)`** — panel-level helper: uses `padLineBackground` for diff lines, plain `lipgloss Width()` for context lines.

### `internal/render/unified.go`
- Compute `contentW` per hunk (`termWidth - 1 - gutterColumns`) to know how much trailing background to add.
- Apply `styledDiffPrefix` for the prefix character on all lines (styled when `LineDiffStyle` active, plain otherwise).
- Apply `padLineBackground` to extend the highlighted content to `contentW` on insert/delete lines.
- Same fix applied to the word-diff pair path.

### `internal/render/split.go`
- Replace `lipgloss.NewStyle().Width(panelWidth).Render(content)` with `renderPanelContent(content, panelWidth, bg)` for both the no-gutter and with-gutter paths.
- Compute `lBg`/`rBg` at the `Split` level (cheap call to `DiffLineStyle`) so panel width extension uses the correct semantic color.
