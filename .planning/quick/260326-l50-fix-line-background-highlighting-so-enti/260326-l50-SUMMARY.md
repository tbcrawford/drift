# Quick Task 260326-l50: Fix Full-Line Background Highlighting

**Date:** 2026-03-26
**Status:** Complete

## What Was Done

Fixed GitHub PR-style full-line background highlighting so that entire delete (red) and insert (green) lines are highlighted, matching terrasort's approach.

## Root Cause (Actual)

Three separate bugs caused the line backgrounds to be completely invisible:

### Bug 1: diffEntryChromaColour returned theme's own background (invisible)
Monokai's `GenericDeleted.Background = #272822` — which is monokai's own terminal background color. `diffEntryChromaColour` checked `e.Background.IsSet()` first and returned it immediately. Result: the "diff highlight" was exactly the same color as the terminal background — invisible.

**Fix**: Skip `Background` when it equals `style.Get(chroma.Background).Background` (the theme's base BG). Fall through to `Colour` blending instead, yielding monokai-red-derived `#44162a` (delete) and monokai-green-derived `#323f1b` (insert).

### Bug 2: Default dark theme was monokai (which suffers from Bug 1)
`autoThemeName(isDark=true)` returned `"monokai"`, and `SelectTheme` also defaulted to `"monokai"`. Since Bug 1 made monokai produce invisible backgrounds, ALL dark terminal users saw no background by default.

**Fix**: Switch default dark theme to `"github-dark"` (same as terrasort). github-dark has explicit diff-aware colors (#490202 delete, #0f5323 insert).

### Bug 3: Prefix character and trailing whitespace had no background
The `+`/`-` prefix and trailing whitespace after highlighted tokens were unstyled. Chroma's `\033[0m` reset at each token boundary prevented lipgloss `Width()` padding from carrying the background.

**Fix**: `styledDiffPrefix` applies background to the prefix; `padLineBackground` measures visible width and appends explicit background-colored spaces to fill the content area width.

## Fix

Followed terrasort's `renderContent` approach:

### New helpers in `internal/render/wordline.go`
- **`styledDiffPrefix(op, bg)`** — returns the `+`/`-`/` ` prefix with the line background applied when `bg` is set.
- **`padLineBackground(content, width, bg)`** — measures `lipgloss.Width(content)`, then appends explicitly background-colored spaces to fill the remaining `width`. This is the key fix for trailing whitespace.
- **`renderPanelContent(content, panelWidth, bg)`** — panel-level helper: uses `padLineBackground` for diff lines, plain `lipgloss Width()` for context lines.

### `internal/render/unified.go`
- Compute `contentW` per hunk (`termWidth - 1 - gutterColumns`) to know how much trailing background to add.
- Apply `styledDiffPrefix` for the prefix character on all lines (styled when `LineDiffStyle` active, plain otherwise).
- Apply `padLineBackground` to extend the highlighted content to `contentW` on insert/delete lines.
- Same fix applied to the word-diff pair path.

### `internal/render/split.go`
- Replace `lipgloss.NewStyle().Width(panelWidth).Render(content)` with `renderPanelContent(content, panelWidth, bg)` for both the no-gutter and with-gutter paths.
- Compute `lBg`/`rBg` at the `Split` level (cheap call to `DiffLineStyle`) so panel width extension uses the correct semantic color.
