<objective>
Fix two confirmed visual bugs in `--chrome delta` mode:

1. **Hunk header box never renders when `codeFragment == ""`** ŌĆö DeltaTheme early-returns `""` causing the fallback `@@ -N,n +N,n @@` format to be used. Delta always renders the box, using `ŌĆó N:` when no fragment is present.

2. **Gutter separator does not match delta's format** ŌĆö Delta uses `{old} Ōŗ« {new} Ōöé{content}` (middle sep = ` Ōŗ« `, right border = ` Ōöé`). Drift currently emits `{old} Ōöé{new}{content}`.

Fix is additive: add `GutterSeparators()` to `ChromeTheme` interface, implement in both themes, wire through `RenderConfig`, and apply in both renderers. Fix hunk header by removing the `codeFragment == ""` early return.
</objective>

**Step A ŌĆö Extend ChromeTheme interface (`internal/chrome/chrome.go`):**

Add method to the `ChromeTheme` interface:
```go
// GutterSeparators returns the strings used between and after gutter number columns.
// middleSep is the separator between the old and new number columns (e.g. " Ōöé" or " Ōŗ« ").
// rightBorder is the string appended after the new number column before line content
// (e.g. "" or " Ōöé").
// noColor disables ANSI styling.
GutterSeparators(noColor bool) (middleSep, rightBorder string)
```

Implement on both themes:
```go
// DriftTheme ŌĆö existing behavior: middle sep = " Ōöé", no right border
func (DriftTheme) GutterSeparators(_ bool) (string, string) {
    return gutterColumnSeparator, ""
}

// DeltaTheme ŌĆö delta format: middle sep = " Ōŗ« ", right border = " Ōöé"  
func (DeltaTheme) GutterSeparators(noColor bool) (string, string) {
    if noColor {
        return " : ", " |"   // ASCII approximation for noColor
    }
    // Styled: Ōŗ« = U+22EE VERTICAL ELLIPSIS, Ōöé = U+2502
    return " Ōŗ« ", " Ōöé"
}
```

Wait ŌĆö re-read the problem statement: delta's noColor gutter separator is not specified as ASCII; the visual bug is about the structure (Ōŗ« + Ōöé), which only displays when color is on. For noColor, use the same character since it's a content separator not a color. Use literal `" Ōŗ« "` and `" Ōöé"` for both color/noColor paths in DeltaTheme (the characters render fine without ANSI; they just won't be colorized).

Actually for DriftTheme noColor, `styledGutterColumnSeparator` already returns the plain `gutterColumnSeparator` string (` Ōöé`), so the styled version wraps it in lipgloss. To keep parity:
- `DriftTheme.GutterSeparators(noColor bool)` returns `(" Ōöé", "")` always (the coloring is handled separately by `styledGutterColumnSeparator`)
- `DeltaTheme.GutterSeparators(noColor bool)` returns `(" Ōŗ« ", " Ōöé")` always

**Step D ŌĆö Apply GutterRightBorder in unified renderer (`internal/render/unified.go`):**

In the `Unified` function, update all `fmt.Fprintf(w, "%s%s%s%s\n", goLeft, gutterSep, goRight, code)` calls (there are 4 of them ŌĆö the two in the word-diff pair branch and the two in the regular branch) to:

```go

```

Also update the `contentW` calculation ŌĆö currently uses `const gutterSepWidth = 2` for `" Ōöé"`. With delta's right border ` Ōöé` added after the new column, the total gutter overhead is:
- DriftTheme: `oldW + len(" Ōöé") + newW + len("")` = `oldW + 2 + newW`
- DeltaTheme: `oldW + len(" Ōŗ« ") + newW + len(" Ōöé")` = `oldW + 3 + newW + 2`

The `contentW` currently hardcodes `gutterSepWidth = 2`. Change to:
```go
gutterSepWidth := len(cfg.GutterMiddleSep) + len(cfg.GutterRightBorder)
if gutterSepWidth == 0 {
    gutterSepWidth = 2 // default " Ōöé"
}
```
Note: `gutterSepWidth` should be computed from the *display widths* of the separators. Since ` Ōŗ« ` is 3 runes and `Ōŗ«` is U+22EE (width 1), `utf8.RuneCountInString(" Ōŗ« ")` = 3 and `utf8.RuneCountInString(" Ōöé")` = 2. However, the existing code uses `len()` (byte count) for this constant, and the diff between byte count and rune count only matters for wide CJK. For simplicity, keep using `len()` here ŌĆö it's a content padding heuristic, off-by-a-few doesn't cause correctness issues.

Rename the const `gutterSepWidth = 2` to a `var` computed from cfg fields.

**Step E ŌĆö Apply GutterRightBorder in split renderer (`internal/render/split.go`):**

The split renderer uses `gutterSep` as the panel separator (between left and right panels via `lipgloss.JoinHorizontal`), NOT as a per-line intra-gutter separator. The left panel's gutter numbers are already `leftG = GutterNumberRender(...)` and joined horizontally with the panel content.

For split view, the `GutterRightBorder` should be applied differently:
- The middle panel separator (`gutterSep` in the `JoinHorizontal` call) stays as `styledGutterColumnSeparator(cfg)` ŌĆö this is the visual divider between left and right panels

**Decision**: For split view, do NOT apply `GutterRightBorder` ŌĆö the split layout already uses `gutterSep` as the panel separator and the panels are self-contained. Only unified mode needs the right border. Add a comment in split.go noting this:

```go

```

Note: `withGutterSeparators` is lowercase (unexported) but lives in the `drift` package. Since `flags.go` is in `package main` and imports `"github.com/tbcrawford/drift"`, it cannot call unexported drift functions. 

**Alternative wiring**: Add `GutterMiddleSep` and `GutterRightBorder` to `rootOptions` and pass directly to `RenderConfig` in `runRoot`/`runPagerMode` where `rcfg` is constructed, bypassing the `drift.Option` pipeline entirely.

Looking at the codebase: `flags.go` builds `driftOpts []drift.Option` and passes them to `drift.Render()`. The chrome gutter config needs to reach `render.RenderConfig` which is built inside `drift.Render`. The cleanest path without exposing new public API:

**Export a `WithGutterSeparators` option** (public API):
```go
// WithGutterSeparators configures the gutter column separator strings.
// middleSep is placed between the old and new line-number columns.
// rightBorder is placed after the new line-number column and before line content.
// Both default to "" which preserves the built-in defaults (" Ōöé" middle, "" right border).
// This is used internally by chrome themes; most callers do not need to set this directly.
func WithGutterSeparators(middleSep, rightBorder string) Option {
    return func(c *config) {
        c.render.gutterMiddleSep = middleSep
        c.render.gutterRightBorder = rightBorder
    }
}
```

Then in `flags.go`:
```go
middleSep, rightBorder := chromeTheme.GutterSeparators(flags.noColor)
opts = append(opts, drift.WithGutterSeparators(middleSep, rightBorder))
```
