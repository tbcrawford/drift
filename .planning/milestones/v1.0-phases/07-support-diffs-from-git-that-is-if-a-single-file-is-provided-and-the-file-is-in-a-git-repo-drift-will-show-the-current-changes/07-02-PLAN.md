## Objective

Wire **one positional argument** into the existing CLI: when the user passes exactly one path (and does not use `--from`/`--to`), resolve **OLD/NEW** via `resolveGitWorkingTreeVsHEAD` from plan 07-01; when two arguments are passed, keep current behavior. Update Cobra argument validation and extend `input` tests so two-path and flag modes stay intact.

## must_haves

- [ ] `drift <single-file>` inside a Git worktree runs diff + render like two-file mode
- [ ] `drift <single-file>` outside a Git worktree fails with usage hint containing `two` and `git`
- [ ] `drift --from x --to y` and `drift a b` unchanged
- [ ] `go test ./cmd/drift/...` exits 0
