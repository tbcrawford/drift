## Objective

Add a small `cmd/drift` package file that, given an absolute filesystem path to a regular file inside a Git worktree, loads **OLD** string from `git show HEAD:<relpath>` (or empty if the path has no `HEAD` blob) and **NEW** from `os.ReadFile`, using `os/exec` with `GIT_TERMINAL_PROMPT=0`. No new module dependencies. This plan does **not** change `main.go` or `input.go`‚Äîonly the helper + tests.

## must_haves

- [ ] `gitworktree.go` exports `resolveGitWorkingTreeVsHEAD(absFile string) (old, new, oldName, newName string, err error)` (name may be unexported if only `package main` uses it‚Äîthen use `resolveGitWorkingTreeVsHEAD` as lowercase `resolveGitWorkingTreeVsHEAD` in same package only)
- [ ] Repo detection via `git -C <dir> rev-parse --is-inside-work-tree` and `--show-toplevel`
- [ ] `HEAD` blob via `git -C <repoRoot> show HEAD:<relpathSlash>`; `relpathSlash` = `filepath.ToSlash(rel)` where `rel` is `filepath.Rel(repoRoot, absFile)`
- [ ] If `git show` fails because path is not in `HEAD`, **OLD = `""`** and still read working tree for NEW (uncommitted new file case)
- [ ] Clear errors when not in a repo or `git` missing: message contains substring `git` and substring `two` (for CLI layer to reuse wording)
- [ ] `go test ./cmd/drift/...` exits 0

**Step 1:** Add `cmd/drift/gitworktree.go` in package `main`:

- Implement `func resolveGitWorkingTreeVsHEAD(path string) (old, new string, oldName, newName string, err error)` as an **unexported** function in package `main`; tests live in `gitworktree_test.go` with `package main` so they call it directly.

- **Abs path:** `filepath.Abs(filepath.Clean(path))` at start; if `os.Stat` is not regular file (or not exist), return error whose message contains `invalid file`.

- **Git dir for rev-parse:** use `filepath.Dir(absFile)` as `-C` target for `rev-parse` calls (per RESEARCH).

- **Commands:** spawn `git` with `Env` = `append(os.Environ(), "GIT_TERMINAL_PROMPT=0")`, `Stdout`/`Stderr` captured.

- **Not inside worktree:** if `rev-parse --is-inside-work-tree` output (trimmed) is not `true`, return `fmt.Errorf("not a git worktree: use two paths to diff files; %w", errOptional)` ensuring final string still contains substrings `git` and `two`.

- **Repo root:** `git -C <dir> rev-parse --show-toplevel` ‚Üí trim newline.

- **Rel path:** `filepath.Rel(repoRoot, absFile)`; if error or `strings.HasPrefix(rel, "..")`, return error containing `outside repository`.

- **Read NEW:** `os.ReadFile(absFile)` ‚Üí string.

- **Read OLD:** run `git -C <repoRoot> show HEAD:` + `relpathSlash`. On failure: if stderr contains `exists on disk, but not in` OR `did not match any file` OR `fatal: path` (choose **documented** substrings from real Git 2.30+), set `old = ""`; else return wrapped error.

- **Names:** `base := filepath.Base(absFile)`; set `oldName = base + " (HEAD)"`, `newName = base + " (working tree)"` exactly (ASCII parentheses and space).

**Step 2:** Add `type gitRunner` or package-level `lookPath = exec.LookPath` only if needed for tests; prefer testing through a temp fake `git` on `PATH` without interfaces if simpler.
</action>
