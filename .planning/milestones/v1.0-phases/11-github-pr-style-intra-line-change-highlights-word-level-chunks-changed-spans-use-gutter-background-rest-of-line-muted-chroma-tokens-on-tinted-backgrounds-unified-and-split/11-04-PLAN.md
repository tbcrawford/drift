# Plan 11-04 — Gap closure: brighter semantic word spans vs muted full-line (UAT)

## Objective

Close the gap in `11-UAT.md`: **intra-line changed spans** must use **semantic remove/add colour** (red vs green), **more saturated / brighter** than the **full-line** background, which stays the **muted** semantic plane. **Unchanged** tokens stay **muted** (comment-style foreground). **`internal/worddiff`** segment ranges are unchanged. Applies to **unified** and **split** word-diff paths. This **supersedes** the 11-03 / terrasort assumption of **neutral gutter-only** word highlights for changed spans.

## must_haves

1. Full-line wrap (`ApplyDiffLineStyle` / `splitApplyDiffLine`) uses a **muted** variant of the theme diff line colour (derived from `DiffLineBackgroundColour`, blended further toward terminal base), so the line wash is visibly softer than word-level emphasis.
2. Changed word spans use a **brighter** semantic background (delete → stronger red family, insert → stronger green family) derived from the same Chroma style + `isDark` + side, **not** `GutterBackgroundHex` / neutral gray.
3. Ordering preserved: segment Chroma → apply word-span background on changed segments → wrap entire rendered line with muted full-line style (prefix + body).
4. `go test ./...` and `just lint` pass; `11-VERIFICATION.md` and UAT gap reference updated.
