## Objective

Introduce `drift.New()` returning a chainable builder whose methods append the same `Option` values as `WithAlgorithm`, `WithContext`, `WithNoColor`, `WithLang`, `WithTheme`, and `WithSplit`, and whose `Diff` / `Render` / `RenderWithNames` methods delegate to package-level `Diff`, `Render`, and `RenderWithNames`. Prove parity with the functional API so CORE-05 and ROADMAP success criterion #1 are satisfied.

## must_haves

- [ ] `drift.New().Algorithm(drift.Histogram).WithTheme("github").Diff(a, b)` compiles and behaves identically to `drift.Diff(a, b, drift.WithAlgorithm(drift.Histogram), drift.WithTheme("github"))` for representative inputs
- [ ] Builder exposes a method for every public `With*` option (same semantics)
- [ ] `go test ./...` exits 0

<action>
**Step 1:** Add `builder.go` in package `drift` with:

- Unexported struct, e.g. `type builder struct { opts []Option }` (exact name unexported) OR exported `type Builder struct { opts []Option }` — prefer **exported `Builder`** if `New` returns `*Builder` and methods use pointer receiver.
- `func New() *Builder` returning empty option slice (no preallocation required).
- Methods on `*Builder`, each returning `*Builder`:
  - `Algorithm(a Algorithm) *Builder` → append `WithAlgorithm(a)`
  - `Context(n int) *Builder` → append `WithContext(n)`
  - `NoColor() *Builder` → append `WithNoColor()`
  - `Lang(lang string) *Builder` → append `WithLang(lang)`
  - `Theme(theme string) *Builder` → append `WithTheme(theme)`
  - `Split() *Builder` → append `WithSplit()`
- Terminal methods:
  - `func (b *Builder) Diff(old, new string) (DiffResult, error)` → `return Diff(old, new, b.opts...)`
  - `func (b *Builder) Render(result DiffResult, w io.Writer) error` → `return Render(result, w, b.opts...)`
  - `func (b *Builder) RenderWithNames(result DiffResult, w io.Writer, oldName, newName string) error` → `return RenderWithNames(result, w, oldName, newName, b.opts...)`

**Step 2:** Add `builder_test.go` with:

- `TestBuilderDiffParity`: same `old`/`new` as a table case; compare `DiffResult.Hunks` length and a stable serialization (e.g. `fmt.Sprintf("%+v", result.Hunks)` or compare `IsEqual` + hunk count) between `New().Algorithm(Patience).Theme("monokai").Diff(old,new)` and `Diff(old,new, WithAlgorithm(Patience), WithTheme("monokai"))`.
- `TestBuilderRenderParity`: `Diff` once; render with `bytes.Buffer` using `WithNoColor()` on both paths; `bytes.Equal` on buffer contents.

**Step 3:** Update `doc.go` package comment to mention `New()` / builder as an alternative to bare `Diff`/`Render` (one sentence).

**Step 4:** Run `go fmt ./...` and `go test ./...`.
</action>
