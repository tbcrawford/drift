## Objective

Add `examples/basic/main.go` and `examples/builder/main.go` as standalone `main` packages that print a visible unified diff to stdout, demonstrating functional vs builder APIs respectively. Satisfy OSS-03 and ROADMAP success criterion #3.

## must_haves

- [ ] `go run ./examples/basic` from module root exits 0 and stdout contains both `-` and `+` diff line prefixes or unified hunk header `@@`
- [ ] `go run ./examples/builder` exits 0 with visible diff output
- [ ] Both examples call `drift.WithNoColor()` or set `NO_COLOR=1` in code comments and use `WithNoColor()` in the example so raw log output is deterministic

## Tasks

<task id="06-03-01" title="examples/basic ‚Äî Diff + Render">
<read_first>
- drift.go ‚Äî `Diff` usage
- render.go ‚Äî `Render` usage
- doc.go ‚Äî package import path
</read_first>

<action>
Create `examples/basic/main.go` with `package main`:

- `import` stdlib `fmt`, `os` and `github.com/tylercrawford/drift`
- Define two multi-line strings `old` and `new` that differ by at least one line (e.g. version bump comment)
- `result, err := drift.Diff(old, new)`; handle error
- `err = drift.Render(result, os.Stdout, drift.WithNoColor())`; handle error
- Optional: `drift.WithLang("go")` if content looks like Go code

Keep total file under ~60 lines.
</action>

- `go run ./examples/basic 2>&1 | grep -qE '@@|^-|^\+'` exits 0 (at least one match)
- `go run ./examples/basic` exits 0
</acceptance_criteria>
</task>
