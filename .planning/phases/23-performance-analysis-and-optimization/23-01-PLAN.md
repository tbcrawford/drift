<objective>
Establish a CPU baseline via profiling, then eliminate the dominant gutter rendering
allocation hotspot: `gutterStyleForCell` currently constructs a new `lipgloss.NewStyle()`
on every line in every hunk. A `RenderConfig`-scoped cache reduces this to a map lookup.

Purpose: The profiling data shows `GutterNumberRender` accounts for 10% of cumulative
CPU and `lipgloss.Style.Render` for 14.35%. Both trace directly to `gutterStyleForCell`
constructing a new `lipgloss.Style` per line. Additionally, `styledGutterColumnSeparator`
is called once per row but always produces the same string; pre-computing it eliminates
redundant lipgloss renders throughout a hunk.

Output: Extended benchmark suite + gutter style cache + pre-computed separator string
that measurably reduces ns/op and allocs/op in render benchmarks.
</objective>

**The hotspot:** `gutterStyleForCell` calls `lipgloss.NewStyle()` + multiple `.Background()` /
`.Foreground()` method chains on every call. With 10k lines there are ~20k calls per render pass.

**The fix: add a `gutterStyleKey` + cache to `RenderConfig`.**

Step 1 — Add a style cache type to `gutter.go`:
```go
// gutterStyleKey uniquely identifies a gutter cell style variant.
// There are at most 6 distinct styles per render call (2 sides × 3 ops).
type gutterStyleKey struct {
    oldColumn bool
    op        edittype.Op
}

// GutterStyleCache pre-computes all gutter cell styles for a render call.
// Build it once at the start of a hunk render loop; reuse for every line.
type GutterStyleCache struct {
    styles  map[gutterStyleKey]lipgloss.Style
    noColor bool
}

// NewGutterStyleCache builds a GutterStyleCache from the resolved render config.
// Call once per Render invocation (not per hunk, not per line).
func NewGutterStyleCache(style *chroma.Style, isDark, noColor bool) *GutterStyleCache {
    // pre-populate all 6 combinations
    cache := &GutterStyleCache{
        styles:  make(map[gutterStyleKey]lipgloss.Style, 6),
        noColor: noColor,
    }
    for _, oldCol := range []bool{true, false} {
        for _, op := range []edittype.Op{edittype.Equal, edittype.Delete, edittype.Insert} {
            k := gutterStyleKey{oldColumn: oldCol, op: op}
            cache.styles[k] = gutterStyleForCell(style, isDark, noColor, oldCol, op)
        }
    }
    return cache
}

// Get returns the pre-computed style for the given cell.
func (c *GutterStyleCache) Get(oldColumn bool, op edittype.Op) lipgloss.Style {
    return c.styles[gutterStyleKey{oldColumn: oldColumn, op: op}]
}
```

Step 2 — Add `GutterCache *GutterStyleCache` field to `RenderConfig` in `gutter.go`.
