  <action>
Rewrite main.go following the ARCHITECTURE.md canonical lifecycle. Preserve all existing
behaviour and test surface. Key changes:

**Remove:**
- `var stdinReader io.Reader = os.Stdin` (global mutable state)
- `var rootCmd = &cobra.Command{...}` (global cobra command)
- `func init()` flag registration block
- `buildDriftOptions(cmd *cobra.Command)` function (replaced by resolveRootOptions in flags.go)
- Direct `fmt.Fprintf(os.Stderr, ...)` in show-theme callback (now uses streams.Err in flags.go)

**Add:**
- `func newRootCmd(streams IOStreams) *cobra.Command` that:
  1. Creates a local `flags := &rootFlags{}` 
  2. Creates the cobra.Command with `Args: cobra.ArbitraryArgs` (remove ValidateArgs from struct init; do validation in RunE via resolveRootOptions)
  3. Registers all flags using `cmd.Flags()` binding to `flags` struct fields (same flags as current init() block)
  4. Sets `RunE` as a two-liner: `opts, err := resolveRootOptions(flags, streams, args)` then `return runRoot(opts)`
  5. Returns the cmd

- `func runRoot(opts *rootOptions) error` that:
  1. Calls `resolveInputs(opts.args, opts.from, opts.to, opts.streams.In)` 
  2. Calls `drift.Diff(old, newText, opts.driftOpts...)`
  3. If `result.IsEqual`, returns nil
  4. Calls `drift.RenderWithNames(result, opts.streams.Out, oldName, newName, opts.driftOpts...)`
  5. Returns `newExitCode(1, "")` on success with differences

- Update `runCLI(streams IOStreams, args []string) int` signature to accept an IOStreams (drop
  the separate stdout/stderr/stdin params). Internally:
  1. Creates `cmd := newRootCmd(streams)` each invocation (no shared global)
  2. `cmd.SetArgs(args)`
  3. `err := cmd.Execute()` 
  4. Error handling unchanged (exitCodeErr unwrap)

- Update `executeDrift() int` to call `runCLI(System(), os.Args[1:])`.

**Validate args inline:** Move the `validateRootArgs` logic into `resolveRootOptions` (already
handles --from/--to + args conflicts). The cobra command uses `Args: cobra.ArbitraryArgs` and
validation happens in resolveRootOptions before any work begins.

**Important test compatibility:**
- `runCLI` signature changes — update callers in main_test.go and any other test that calls
  `runCLI(stdout, stderr, stdin, args)` to use `runCLI(IOStreams{In: stdin, Out: stdout, Err: stderr}, args)`
- `TestHelpListsAllFlags` in main_test.go accesses `rootCmd` directly — update it to call
  `newRootCmd(System())` locally, or restructure the test to use `runCLI` with `--help` args
- `cli_test.go` uses `testscript.Main` + `executeDrift()` — this still works since executeDrift()
  calls `runCLI(System(), ...)` unchanged externally

**Preserve `parseAlgorithm`** — it is reused by resolveRootOptions in flags.go.

No business logic changes. No new drift.Option types. No rendering changes. This is purely
structural CLI wiring.
