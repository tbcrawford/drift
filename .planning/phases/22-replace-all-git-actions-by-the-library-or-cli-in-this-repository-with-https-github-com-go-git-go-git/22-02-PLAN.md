6. Returns the repo dir

- Write a `.gitignore` file with the pattern before committing

- Create a repo with a `.gitignore` containing `dist/` (or `dist/app`)
- Write `dist/app` to disk (it may or may not be committed — gitignore matches regardless)
- Call `filterGitIgnored(repoDir, []string{"main.go", "dist/app", "README.md"})`
- Verify `dist/app` is removed from the result

For `TestGitDirectoryVsHEAD_skipsIgnored`:
- Init repo, write and commit `keep.go` with content "oldcontent"
- Add `dist/` to `.gitignore` (commit the .gitignore)
- Modify `keep.go` content to "changed" (do NOT stage/commit)
- Write `dist/app` to disk (not staged)
- Call `gitDirectoryVsHEAD(repoDir)`
- Verify only `keep.go` appears in pairs (dist/app ignored)
- Verify `keep.go` pair has HeadContent="oldcontent" and WorkContent="changed"

For `TestResolveGitWorkingTreeVsHEAD_happyPath`:
- Init repo, write `file.txt` = "oldcontent", stage + commit
- Overwrite `file.txt` = "new" (do NOT stage/commit)
- Call `resolveGitWorkingTreeVsHEAD(filepath.Join(repoDir, "file.txt"))`
- Verify old="oldcontent", new="new", oldName ends with "(HEAD)", newName ends with "(working tree)"

For `TestResolveGitWorkingTreeVsHEAD_notInRepo`:
- Create a plain temp dir (no git init)
- Write a file there
- Call `resolveGitWorkingTreeVsHEAD` on it
- Verify error message contains words about git and "two paths" (or equivalent)

For `TestResolveGitWorkingTreeVsHEAD_missingHEADBlob`:
- Init repo (no files committed — HEAD doesn't exist, or commit with different file)
- Write `file.txt` to disk (untracked)
- Call `resolveGitWorkingTreeVsHEAD` → old="" (not in HEAD), new=disk content

`gitShowHEADBlob` — if Head() fails, the file definitely doesn't exist at HEAD, return "".

blob as empty string, not an error.
