<objective>
Create `.goreleaser.yaml` at the repository root so that `goreleaser release`
(or `goreleaser release --snapshot --clean` for local testing) produces
multi-platform binaries for the `drift` CLI.

Target platforms:
- darwin/amd64 (Intel Mac)
- darwin/arm64 (Apple Silicon Mac)
- linux/amd64 (CI/CD, Linux servers)
- windows/amd64 (Windows x64)

The CLI binary lives at `cmd/drift/` — goreleaser's `main` field for the build
must point there. The binary name is `drift`.

The config should follow goreleaser v2 schema (the current major version as of
2025). Key sections needed:
- `project_name`
- `builds` — one entry with goos/goarch matrix, ldflags for version injection
- `archives` — tar.gz for non-Windows, zip for Windows
- `checksum` — sha256
- `changelog` — optional but include a basic `sort: asc` config

ldflags should inject version info via:
  `-s -w -X main.version={{.Version}} -X main.commit={{.Commit}} -X main.date={{.Date}}`

This allows the CLI to optionally expose version info via `drift --version`
in a future release. The variables may not be declared in main.go yet — that
is fine; goreleaser will link them as undefined-but-harmless (linker ignores
unknown -X targets in Go).
</objective>

  <action>
Create `.goreleaser.yaml` at the repository root with the following content:

```yaml
# .goreleaser.yaml — goreleaser v2 configuration for drift CLI
# https://goreleaser.com/customization/
version: 2

project_name: drift

before:
  hooks:
    - go mod tidy
    - go generate ./...

builds:
  - id: drift
    main: ./cmd/drift
    binary: drift
    env:
      - CGO_ENABLED=0
    goos:
      - linux
      - darwin
      - windows
    goarch:
      - amd64
      - arm64
    # Exclude linux/arm64 and windows/arm64 for the initial release.
    # These can be added once CI has arm64 runners.
    ignore:
      - goos: windows
        goarch: arm64
      - goos: linux
        goarch: arm64
    ldflags:
      - -s -w
      - -X main.version={{.Version}}
      - -X main.commit={{.Commit}}
      - -X main.date={{.Date}}

archives:
  - id: drift-archive
    formats:
      - tar.gz
    # Use zip for Windows releases (conventional for the platform).
    format_overrides:
      - goos: windows
        formats:
          - zip
    name_template: "{{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}"
    files:
      - LICENSE
      - README.md

checksum:
  name_template: "{{ .ProjectName }}_{{ .Version }}_checksums.txt"
  algorithm: sha256

snapshot:
  # Version string used when running --snapshot (local testing without a tag).
  version_template: "{{ .Tag }}-next"

changelog:
  sort: asc
  filters:
    exclude:
      - "^docs:"
      - "^test:"
      - "^chore:"
      - Merge pull request
      - Merge branch
```

Note: goreleaser v2 changed the `snapshot.name_template` key to
`snapshot.version_template`. Use `version_template` to avoid deprecation warnings.
