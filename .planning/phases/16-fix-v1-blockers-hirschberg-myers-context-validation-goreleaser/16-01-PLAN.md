<objective>
Replace the O((N+M)²) trace-snapshot Myers implementation with a Hirschberg
linear-space divide-and-conquer variant that stores only O(N+M) state at any
point in the recursion.

Current problem: `myers.go` keeps a `trace [][]int` where trace[d] is a full
copy of the V array. Each copy is O(N+M) and there are O(N+M) edit-distance
steps, giving O((N+M)²) peak memory. For a 10,000-line file this is ~800 MB.

Solution: Hirschberg's insight (adapted to Myers by Hirschberg 1975 / Myers
1986 §4) — to find the edit path between (0,0)→(N,M), run the forward pass
only to d = ⌊(N+M)/2⌋ and the reverse pass (starting from (N,M)) for the same
budget. The two passes must overlap on at least one diagonal. The overlapping
point is the "midpoint" of the optimal path. Recurse on the two halves. This
uses only two V arrays (forward and reverse, each O(N+M)) and a call stack
O(N+M) deep — total O(N+M) space. Time complexity remains O(ND).

The public interface — `Myers.Diff(oldLines, newLines []string) []edittype.Edit`
— does not change. All callers continue to work without modification.
</objective>

**Algorithm structure:**

```
Myers.Diff(old, new) []Edit
  → handles edge cases (empty inputs) → returns directly
  → calls diff(old, new, oldOff, newOff) which appends to edits

diff(old, new, oldOff, newOff int, edits *[]Edit)
  N, M := len(old), len(new)
  if N == 0 → append M inserts; return
  if M == 0 → append N deletes; return
  if N == 1 && M == 1 → equal or delete+insert; return
  mx, my := midpoint(old, new)      // O(N+M) space, O(ND) time
  diff(old[:mx], new[:my], oldOff, newOff, edits)
  diff(old[mx:], new[my:], oldOff+mx, newOff+my, edits)
```

**midpoint(old, new []string) (mx, my int)**

The midpoint function finds the split point of the optimal edit path:

1. Allocate two V arrays: `vf` (forward, size 2*(N+M)+1) and `vr` (reverse, same size).
2. `delta = N - M`
3. Forward: iterate d = 0..max; for k in -d..d step 2, extend diagonal k as far as possible; save vf[k].
4. Reverse: iterate d = 0..max; for k in -d..d step 2 (offset by delta), extend diagonal k (going backwards) as far as possible; save vr[k].
5. After each d step, check if forward and reverse overlap on any diagonal: overlap condition is `(delta+d) % 2 != 0` (odd total edit distance) for forward checking reverse, or `(delta+d) % 2 == 0` for reverse checking forward. The standard check: after advancing forward to d, check if any k in the reverse's band overlaps; vice versa.
6. The overlap point (x, y) is the midpoint. Return it as (x, y).

**Detailed midpoint algorithm (proven correct):**

```go
func midpoint(old, new []string) (int, int) {
    N, M := len(old), len(new)
    max := N + M
    // vf[k+max] = furthest x on diagonal k going forward
    // vr[k+max] = furthest x from (N,M) on diagonal k going backward
    vf := make([]int, 2*max+1)
    vr := make([]int, 2*max+1)
    vf[1+max] = 0
    vr[1+max] = 0

    delta := N - M

    for d := 0; d <= (max+1)/2; d++ {
        // Forward step
        for k := -d; k <= d; k += 2 {
            var x int
            if k == -d || (k != d && vf[k-1+max] < vf[k+1+max]) {
                x = vf[k+1+max]
            } else {
                x = vf[k-1+max] + 1
            }
            y := x - k
            for x < N && y < M && old[x] == new[y] {
                x++
                y++
            }
            vf[k+max] = x
            // Check overlap with reverse: reverse diagonal k_r = k - delta
            // The reverse V is indexed from (N,M) going back; overlap when
            // forward x + reverse_x_from_end >= N (i.e. they've met or crossed)
            if (delta % 2 != 0) && (k-delta >= -(d-1)) && (k-delta <= d-1) {
                // vr stores distance from N: actual x = N - vr[...], y = M - (x-k_r)
                // k_r = k - delta
                kr := k - delta
                if vf[k+max]+vr[kr+max] >= N {
                    // midpoint is at (vf[k+max], vf[k+max]-k)
                    return vf[k+max], vf[k+max] - k
                }
            }
        }

        // Reverse step
        for k := -d; k <= d; k += 2 {
            // reverse diagonal is k+delta in the original (old,new) space
            var x int
            if k == -d || (k != d && vr[k-1+max] < vr[k+1+max]) {
                x = vr[k+1+max]
            } else {
                x = vr[k-1+max] + 1
            }
            y := x - k
            // Extend snake backward: from (N-x, M-y) while lines match
            for x < N && y < M && old[N-x-1] == new[M-y-1] {
                x++
                y++
            }
            vr[k+max] = x
            // Check overlap with forward: forward diagonal kf = k + delta
            if (delta % 2 == 0) && (k+delta >= -d) && (k+delta <= d) {
                kf := k + delta
                if vf[kf+max]+vr[k+max] >= N {
                    // midpoint: forward reached vf[kf+max] on diagonal kf
                    fx := vf[kf+max]
                    fy := fx - kf
                    return fx, fy
                }
            }
        }
    }

    // Fallback: should not be reached for valid inputs
    return N / 2, M / 2
}
```

**Important implementation notes:**

1. The recursive `diff` helper takes `oldOff, newOff int` offset parameters so
   Edit line numbers are computed as `oldOff + localIdx + 1` (1-indexed).

2. Base cases in `diff`:
   - `N == 0, M == 0`: return immediately (no edits)
   - `N == 0`: emit M Insert edits at newOff+1..newOff+M
   - `M == 0`: emit N Delete edits at oldOff+1..oldOff+N
   - `N == 1, M == 1`: if `old[0] == new[0]` emit Equal; else emit Delete then Insert

3. The edits slice is built in a single pre-allocated buffer passed by pointer
   through all recursive calls. Final output is already in forward order — no
   reversal needed.

4. `copyV` helper is no longer needed; remove it.

5. The outer `Diff` method allocates `edits := make([]edittype.Edit, 0, N+M)`
   and passes `&edits` to the recursive `diff` helper.

6. Line numbers in emitted edits are 1-indexed (same as before):
   - Equal: `OldLine: oldOff+x+1, NewLine: newOff+y+1` for each step along the snake
   - Insert: `OldLine: 0, NewLine: newOff+y+1`
   - Delete: `OldLine: oldOff+x+1, NewLine: 0`

**Test 1: TestHirschbergMemory** — verifies that Myers.Diff does NOT allocate
O(N²) memory on a large input. Uses `testing.AllocsPerRun` to check that
allocation count grows linearly with input size, not quadratically. Concretely:

```go
// TestHirschbergMemory verifies that Myers.Diff allocates O(N+M) — not O((N+M)²).
// The old trace-snapshot approach allocated O(d) V-array copies, each O(N+M) ints,
// totaling O((N+M)²) for a fully-differing input. The Hirschberg variant recurses
// with two O(N+M) V arrays and no per-step snapshots.
//
// Verification: allocs for 200-line input should be < 10× allocs for 100-line input.
// (Quadratic growth would give ~4× allocs for 2× input; linear gives ~2×.)
// We use a generous 10× bound to be robust across Go versions.
func TestHirschbergMemory(t *testing.T) {
    m := myers.New()

    // Build two completely-different inputs (worst-case edit distance = N+M).
    make100 := func(prefix string, n int) []string {
        lines := make([]string, n)
        for i := range lines {
            lines[i] = fmt.Sprintf("%s-line-%d", prefix, i)
        }
        return lines
    }

    old100 := make100("old", 100)
    new100 := make100("new", 100)
    old200 := make100("old", 200)
    new200 := make100("new", 200)

    allocs100 := testing.AllocsPerRun(5, func() { m.Diff(old100, new100) })
    allocs200 := testing.AllocsPerRun(5, func() { m.Diff(old200, new200) })

    ratio := allocs200 / allocs100
    if ratio > 10 {
        t.Errorf("allocation ratio %.1f exceeds 10× — possible O(N²) growth (allocs100=%.0f allocs200=%.0f)",
            ratio, allocs100, allocs200)
    }
}
```

// TestHirschbergLarge verifies correctness on a 500-line input that exceeds
// the size where trace-snapshot quadratic growth would be most visible.
func TestHirschbergLarge(t *testing.T) {
    m := myers.New()

    old := make([]string, 500)
    new := make([]string, 500)
    for i := range old {
        old[i] = fmt.Sprintf("line %d", i)
    }
    for i := range new {
        if i%7 == 0 {
            new[i] = fmt.Sprintf("changed %d", i)
        } else {
            new[i] = fmt.Sprintf("line %d", i)
        }
    }

    edits := m.Diff(old, new)
    eq, ins, del := countOps(edits)

    if eq+del != len(old) {
        t.Errorf("old invariant: Equal(%d)+Delete(%d)=%d != len(old)=%d", eq, del, eq+del, len(old))
    }
    if eq+ins != len(new) {
        t.Errorf("new invariant: Equal(%d)+Insert(%d)=%d != len(new)=%d", eq, ins, eq+ins, len(new))
    }
}
