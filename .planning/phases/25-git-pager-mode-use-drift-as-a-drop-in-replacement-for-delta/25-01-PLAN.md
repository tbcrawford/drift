// extracted from a multi-file unified diff stream (e.g. git diff output).

    // OldName is the display path from "--- a/..." (e.g. "a/cmd/drift/main.go").

    OldName string

Create `cmd/drift/unifieddiff.go` in `package main`.

**Parser design:**

Use `bufio.Scanner` to scan line by line. State machine with states:
- `stateHeader` — looking for `diff --git` line
- `stateMeta` — inside header block (index, mode, similarity lines)
- `stateHunk` — processing +/- lines within a `@@` hunk

**Algorithm:**
1. When scanner sees `diff --git a/X b/Y`: finish previous file (if any), start new `parsedFileDiff`
2. In `stateMeta`, look for:
   - `--- a/X` → set `OldName = "a/X"`, `Name = "X"` (strip `a/` prefix); if `--- /dev/null` → file is new
   - `+++ b/X` → set `NewName = "b/X"`; if `+++ /dev/null` → file is deleted
   - `Binary files` → set `IsBinary = true`
   - `@@` → transition to `stateHunk` and process this line
3. In `stateHunk`:
   - Line starting with ` ` (space) → append to BOTH old and new builders (context line — strip leading space)
   - Line starting with `-` → append to old builder only (strip leading `-`)
   - Line starting with `+` → append to new builder only (strip leading `+`)
   - Line starting with `\` (no newline at end) → skip (git metadata)
   - Line starting with `@@` → stay in stateHunk (next hunk in same file)
   - Line starting with `diff --git` → finish current file, start new one
4. When scanner is done, flush last file.

**Name extraction:** strip `a/` or `b/` prefix from the path. For renames (`--- a/old`, `+++ b/new`), use the new name as `Name`.

**OldContent / NewContent reconstruction:** Use `strings.Builder` per file. Lines reconstructed WITHOUT the unified diff prefix character. Each line gets a `\n` appended UNLESS it was followed by a `\ No newline at end of file` marker.

**Edge case:** `--- /dev/null` means `OldContent = ""` (file is new). `+++ /dev/null` means `NewContent = ""` (file is deleted). In these cases, do not reconstruct content from the null side.

Write failing tests first (RED), then implement (GREEN). Run `go test ./cmd/drift/ -run TestParseUnifiedDiff -v` after each phase.
