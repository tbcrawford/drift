<objective>
Update the unified and split renderers to emit function context in `@@` hunk headers when `Hunk.FuncName` is non-empty, matching git's format: `@@ -12,7 +12,9 @@ func ParseOptions(args []string)`.

Purpose: This is the visible output change users will see — the data-layer work (Plan 01) populated `Hunk.FuncName`; this plan makes it appear in the rendered diff output.

Output: Updated `internal/render/unified.go`, `internal/render/split.go`, new renderer tests, and updated golden fixtures if impacted.
</objective>

**Step 1: Update internal/render/unified.go**

Find the hunk header formatting block:
```go
// Hunk header: @@ -OldStart,OldLines +NewStart,NewLines @@
header := fmt.Sprintf("@@ -%d,%d +%d,%d @@\n",
    h.OldStart, h.OldLines, h.NewStart, h.NewLines)
```

Replace with:
```go
// Hunk header: @@ -OldStart,OldLines +NewStart,NewLines @@ [FuncName]
var header string
if h.FuncName != "" {
    header = fmt.Sprintf("@@ -%d,%d +%d,%d @@ %s\n",
        h.OldStart, h.OldLines, h.NewStart, h.NewLines, h.FuncName)
} else {
    header = fmt.Sprintf("@@ -%d,%d +%d,%d @@\n",
        h.OldStart, h.OldLines, h.NewStart, h.NewLines)
}
```
