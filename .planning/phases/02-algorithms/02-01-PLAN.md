# Plan 02-01: Implement Patience diff in `internal/algo/patience/`

## Objective
Implement the Patience diff algorithm as an `algo.Differ`-satisfying package. The algorithm anchors on unique lines (appearing exactly once in each file), computes their LCS to form anchors, then recurses on inter-anchor gaps, falling back to Myers when no unique lines exist in a sub-range.

## must_haves
- [ ] `internal/algo/patience/patience.go` exists and `var _ algo.Differ = (*Patience)(nil)` compiles
- [ ] `patience.New().Diff(old, new)` produces correct edits: `Equal+Delete == len(old)` and `Equal+Insert == len(new)` for all test inputs
- [ ] Myers fallback fires without panic on inputs with no unique common lines
- [ ] `go test ./internal/algo/patience/... -race` exits 0

```go
type frame struct {
    os, oe int // old start/end (0-indexed, exclusive end)
    ns, ne int // new start/end (0-indexed, exclusive end)
}
```

**func (p *Patience) Diff(old, new []string) []edittype.Edit** ‚Äî iterative stack implementation:

1. Handle edge cases identically to Myers:
   - Both empty ‚Üí return `[]edittype.Edit{}`
   - old empty ‚Üí return M Insert edits with NewLine=i+1, OldLine=0
   - new empty ‚Üí return N Delete edits with OldLine=i+1, NewLine=0

2. Initialize: `stack := []frame{{0, len(old), 0, len(new)}}`, `edits := make([]edittype.Edit, 0, len(old)+len(new))`

3. Main loop `for len(stack) > 0`: pop from end (`stack[len(stack)-1]`; `stack = stack[:len(stack)-1]`)

4. Per frame `f`:
   a. **Head trim**: advance `f.os` and `f.ns` while `f.os < f.oe && f.ns < f.ne && old[f.os] == new[f.ns]`, emitting `edittype.Edit{Op: edittype.Equal, OldLine: f.os+1, NewLine: f.ns+1}` for each match, then increment both.
   b. **Tail trim**: advance `te := 0` while `te < (f.oe-f.os) && te < (f.ne-f.ns) && old[f.oe-1-te] == new[f.ne-1-te]`, collecting tail Equal edits in a separate slice (they'll be appended after gap processing).
   c. Adjust `f.oe -= te` and `f.ne -= te`.
   d. **Empty check**: if `f.os == f.oe && f.ns == f.ne` ‚Üí append tail edits and continue.
   e. **Old empty (after trim)**: if `f.os == f.oe` ‚Üí emit `f.ne-f.ns` Insert edits with `NewLine = f.ns+i+1, OldLine=0` ‚Üí append tail ‚Üí continue.
   f. **New empty (after trim)**: if `f.ns == f.ne` ‚Üí emit `f.oe-f.os` Delete edits with `OldLine = f.os+i+1, NewLine=0` ‚Üí append tail ‚Üí continue.
   g. **Find unique elements**: build frequency maps for `old[f.os:f.oe]` and `new[f.ns:f.ne]`. Collect lines that appear exactly once in both. Returns `type uniqueEntry struct{ line string; oldIdx, newIdx int }`.
   h. **LCS of unique anchors**: run O(N¬∑M) LCS DP over unique entries. Returns `[]anchor` where `anchor = struct{ oldIdx, newIdx int }` (0-indexed positions in the full slices).
   i. **No anchors ‚Üí Myers fallback**: if `len(anchors) == 0`, call `myers.New().Diff(old[f.os:f.oe], new[f.ns:f.ne])`, apply offsets (`OldLine += f.os` for non-zero OldLine, `NewLine += f.ns` for non-zero NewLine), append to edits, append tail edits, continue.
   j. **With anchors**: process each gap between consecutive anchors + the gaps before first and after last anchor. For each gap `(gos, goe, gns, gne)`, push `frame{gos, goe, gns, gne}` onto stack (push in reverse so leftmost gap is processed first: push right gap then left gap). Emit Equal edits for each anchor: `edittype.Edit{Op: edittype.Equal, OldLine: anchor.oldIdx+1, NewLine: anchor.newIdx+1}`. Then append tail edits.

**Helper functions (unexported):**

`uniqueElements(lines []string, start, end int) map[string]int` ‚Äî returns frequency map for lines[start:end].

`lcsAnchors(oldUniq, newUniq []struct{ line string; idx int }) []struct{ oldIdx, newIdx int }` ‚Äî O(N¬∑M) LCS DP over unique line slices; returns matched anchor pairs with original file indices.

`applyOffset(edits []edittype.Edit, oldOff, newOff int)` ‚Äî mutates slice: adds oldOff to OldLine when OldLine > 0, adds newOff to NewLine when NewLine > 0.

**Edit order:** Process frames left-to-right by pushing the right sub-range onto the stack first, then the left sub-range. Since we pop from the end, the left sub-range is popped and processed first, producing edits in file order without any post-sort step.
